# 조하리 창 결과 분석 설계

## 목표

본인 설문 1건과 피어 설문 3건 이상을 조하리 창 관점으로 해석해, 일관된 전체 결과와 사분면별 텍스트·이미지 결과를 제공한다.

## 입력

- `SELF`: 설문 당사자의 자기 인식 응답 1묶음
- `PEER_1` ~ `PEER_N`: 피어별 응답 묶음. 실제 식별자는 AI에 전달하지 않는다.
- 각 답변에는 질문, 답변, 특성 코드, 응답 출처와 피어 가명 식별자가 포함된다.

## 분석 생성

텍스트 분석은 OpenAI 구조화 응답 호출 1회로 생성한다. 페르소나는 친근하지만 과장하지 않는 한국어 조하리 창 해설자·분석가다.

AI는 자기 인식과 피어 인식을 문맥적으로 비교한다.

- `OPEN`: SELF와 PEER 모두에서 공통으로 드러난 특성
- `BLIND`: PEER에서 두드러지고 SELF에서는 드러나지 않은 특성
- `HIDDEN`: SELF에서 드러나고 PEER에서는 드러나지 않은 특성
- `UNKNOWN`: 강한 간접 근거가 있을 때만 잠재력으로 표현한다. 그렇지 않으면 아직 드러나지 않은 탐색 영역으로 표현한다.

고정된 피어 다수결 규칙은 두지 않는다. 피어별 묶음과 전체 응답 맥락을 AI가 해석한다.

## 생성 결과

### 전체 결과

- 전체 키워드 1개: 예) `마음을 잘 여는 사람`
- 종합 분석 1개
- `이렇게 해보는 건 어때요?` 팁 1개

### 사분면 결과

각 `OPEN`, `BLIND`, `HIDDEN`, `UNKNOWN`에는 다음을 만든다.

- 캐릭터형 정의 키워드 1개: 예) `탐험가`, `천방지축`
- 짧고 말맛 있는 형용사/태그 2개: 예) `탐험 실험 다 좋아 인간`, `새로운 거? 무조건 해봐야지`
- 한국어 해석 1개
- 개인 식별 정보와 원문 답변을 포함하지 않는 추상 이미지 프롬프트 1개
- 해당 프롬프트로 만든 PNG 이미지 1개

구조화 응답은 사분면별 항목을 정확히 한 번씩 반환해야 한다. 방어적으로 중복 사분면은 최초 항목만 보존하며, 사분면이 누락된 경우에는 분석 생성 실패로 처리한다.

## 저장 및 조회

- 분석 텍스트와 이미지 프롬프트는 결과 생성 시 DB에 저장한다.
- 사분면 이미지는 기존과 같이 `surveys/{surveyCode}/results/{quadrant}.png`에 S3 저장한다.
- 결과 조회 API는 전체 결과, 사분면별 키워드·태그·해석·presigned 이미지 URL, 팁을 공통 응답 구조 안에서 반환한다.
- 원천 답변의 제출자 유형은 결과 분석 데이터에도 저장하며, 기존 행은 Flyway 마이그레이션에서 원본 제출 데이터로 백필한다.

## 실패 및 재시도

- 텍스트 분석 생성 실패: 전체 분석 단위로 재시도한다.
- 이미지 생성 실패: 실패한 사분면 이미지만 재시도한다.
- 텍스트 분석은 정상 저장된 뒤 이미지 생성과 독립적으로 재시도할 수 있어야 한다.

## 검증

- SELF 1묶음과 PEER 3묶음의 출처·가명 식별자가 분석 입력에 보존되는지 검증한다.
- AI 구조화 출력에서 전체 결과·팁·사분면별 필수 필드와 중복/누락을 검증한다.
- Flyway 마이그레이션과 결과 조회 API 계약을 통합 테스트한다.
